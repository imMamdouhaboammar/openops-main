## 1. Product-Led Growth (PLG) Strategy {#plg}

### Description

A business methodology where the product itself is the primary driver of customer acquisition, conversion, expansion, and retention. Users discover, adopt, and scale with minimal sales intervention.

### Core PLG Principles

#### 1. Seamless Self-Service Experience

- **Email-Only Sign-Up**: No credit card required for trial/freemium
- **Instant Access**: No approval workflows, no sales calls
- **Progressive Profiling**: Collect data over time, not upfront
- **Guided Onboarding**: Interactive tours, contextual tooltips

#### 2. Time-to-Value (TTV) Optimization

**Goal**: First "Aha!" moment in < 5 minutes

**Tactics**:

- Pre-populated templates/examples
- Smart defaults (no blank canvas syndrome)
- Skip unnecessary setup steps
- Show value before asking for commitment

**Example: Slack**

- Create workspace in 30 seconds
- Pre-loaded with #general and #random channels
- Send first message immediately
- Invite team members later

#### 3. Built-In Virality

Product features that inherently encourage sharing:

**Examples**:

- **Figma**: Multiplayer design → need to invite team
- **Notion**: Shared workspaces → collaboration required
- **Loom**: Video sharing → every view is an impression
- **Calendly**: Booking links → every meeting promotes tool

#### 4. Freemium/Free Trial Model

- **Freemium**: Free forever with limitations (users, features, storage)
- **Free Trial**: Full access for X days, then paywall
- **Reverse Trial**: Start premium, downgrade to free

**Conversion Strategies**:

- Usage limits (storage, team size, features)
- Feature gating (advanced analytics, integrations)
- Time-based triggers (trial ending emails)

#### 5. Product Qualified Leads (PQLs)

Users exhibiting buying intent through product behavior

**PQL Criteria Examples**:

```
Score = 0

// Usage signals
+10 points: Used core feature 3+ times
+15 points: Invited team members
+20 points: Created 5+ projects/documents
+25 points: Hit freemium limit

// Firmographic signals
+15 points: Company size > 50 employees
+10 points: Enterprise email domain
+5 points: Target industry

PQL Threshold: 50+ points
```

### PLG Metrics Dashboard

| Metric | Description | Target |
|--------|-------------|--------|
| **Free-to-Paid Conversion** | % of free users who upgrade | 2-5% (good), 7-10% (excellent) |
| **PQL-to-Customer** | % of PQLs who become paying | 25-30% |
| **Expansion Revenue** | Revenue from existing customers | > 30% of total revenue |
| **Net Revenue Retention (NRR)** | Revenue retention + expansion | > 120% (best-in-class) |
| **Viral Coefficient (K)** | New users brought by existing | > 1 (exponential growth) |

## 2. Viral Loop Design & Engineering {#viral-loops}

### Description

Systematic design of self-reinforcing growth cycles where existing users bring in new users, creating exponential growth.

### Viral Loop Types

#### Type 1: Inherent Virality (Product-Driven)

**Mechanism**: Using the product inherently spreads awareness

**Examples**:

- **Zoom**: Every meeting invite = Zoom ad
- **Calendly**: Every booking link promotes tool
- **DocuSign**: Every signature request exposes brand
- **Typeform**: "Powered by Typeform" footer

**Design Principles**:

- Make sharing core to value delivery
- Reduce friction for non-users (no account required)
- Brand visibility in shared artifacts

#### Type 2: Incentivized Virality (Referral Programs)

**Mechanism**: Rewards for bringing in new users

**Double-Sided Incentive Structure**:

```
Referrer Gets:
- Tier 1: 1 referral = $10 credit
- Tier 2: 5 referrals = $75 credit ($15 each)
- Tier 3: 10+ referrals = $200 credit ($20 each)

Referred User Gets:
- $10 credit on first purchase
- Exclusive onboarding support
```

**Best Practices**:

- Reward both parties (not just referrer)
- Escalating rewards for multiple referrals
- Time-limited bonuses (refer 3 friends this month)
- Non-monetary rewards (exclusive features, early access)

#### Type 3: Social Virality (Sharing as Status)

**Mechanism**: Users share to signal identity or achievement

**Examples**:

- **Spotify Wrapped**: Annual listening summary
- **Strava**: Fitness achievements, route maps
- **LinkedIn**: Course completions, certifications
- **Duolingo**: Streak milestones, leaderboards

**Psychology**:

- **Social Currency**: Makes me look good
- **FOMO**: Others are doing it
- **Self-Expression**: Reflects my identity

#### Type 4: Network Effects Virality

**Mechanism**: Product value increases with more users

**Types**:

- **Direct Network Effects**: WhatsApp (more contacts = more value)
- **Two-Sided Marketplace**: Uber (more riders ↔ more drivers)
- **Data Network Effects**: Waze (more users = better traffic data)

### Viral Coefficient (K-Factor) Optimization

**Formula**:

```
K = (Invites Sent Per User) × (Conversion Rate of Invites)

Example:
Average user sends 5 invites
25% of recipients sign up
K = 5 × 0.25 = 1.25 (Viral! 🚀)
```

**Optimization Levers**:

1. **Increase Invites Sent**:
   - Multiple touchpoints (email, in-app, social)
   - Gamification (leaderboards for most referrals)
   - Incentives (extra storage, credits)

2. **Improve Conversion Rate**:
   - Personalized referral messages
   - Landing page optimization
   - Reduce signup friction
   - Social proof (X friends already using)

### Viral Cycle Time

How long it takes for one viral loop to complete

**Impact**:

- Cycle Time = 7 days, K = 1.2 → 2x growth in ~30 days
- Cycle Time = 2 days, K = 1.2 → 2x growth in ~9 days

**Shortening Cycle Time**:

- Send reminders to invitees
- Immediate value for new users (fast activation)
- Real-time notifications (friend just joined!)

## 3. Customer Lifetime Value (CLV) Optimization {#clv}

### Description

Systematic approach to increasing the total revenue generated from a customer over their entire relationship.

### CLV Calculation Methods

#### Historical CLV (Actual)

```
CLV = Sum of all revenue from customer to date

Example:
Customer signed up: Jan 2024
Subscriptions: 12 months × $50/mo = $600
Upsells: Premium tier (+$20/mo) for 6 months = $120
Total CLV: $720
```

#### Predictive CLV (Forecasted)

```
CLV = (ARPU × Gross Margin) / Churn Rate

Example:
ARPU = $60/month
Gross Margin = 80%
Monthly Churn = 5%

CLV = ($60 × 0.80) / 0.05 = $960
```

### CLV Optimization Strategies

#### 1. Increase ARPU (Average Revenue Per User)

**Tactics**:

- **Pricing Optimization**: Test tier pricing, find sweet spot
- **Upselling**: Move users to higher tiers
- **Cross-Selling**: Add-ons, modules, integrations
- **Usage-Based Pricing**: Align cost with value received

**Example: Upsell Campaign**

```
Trigger: Free user hits 80% of storage limit
Email: "You're running out of space! Upgrade to Pro for unlimited storage"
Offer: 20% off first 3 months
Conversion: 15% of triggered users upgrade
```

#### 2. Increase Purchase Frequency

**Tactics**:

- **Subscription Model**: Monthly/annual recurring revenue
- **Consumables**: Credits, API calls (buy more when depleted)
- **Seasonal Campaigns**: Black Friday, year-end deals

#### 3. Extend Customer Lifespan (Reduce Churn)

**Churn Reduction Playbook**:

**Early Warning System**:

```
Churn Risk Score = 0

// Engagement signals
-20: No login in 14 days
-15: Feature usage declined 50%
-10: Support ticket unresolved

// Satisfaction signals
-30: NPS score < 6
-20: Survey response "considering alternatives"

Threshold: Score < -50 = High Risk
```

**Intervention Strategy**:

- **High Risk (Score < -50)**: Personal outreach from CSM
- **Medium Risk (-30 to -50)**: Automated email with resources
- **Low Risk (-10 to -30)**: In-app engagement nudge

#### 4. Expansion Revenue (Net Revenue Retention > 100%)

**Goal**: Existing customers spend MORE this year than last year

**Tactics**:

- **Seat Expansion**: Team grows, adds more licenses
- **Feature Upsells**: Unlock advanced capabilities
- **Usage-Based Growth**: Natural increase in API calls, storage
- **Cross-Sell**: Additional products/modules

**Example: Slack**

```
Year 1: Team of 10 × $8/user/mo = $80/mo = $960/year
Year 2: Team grows to 25 users = $200/mo
Year 2: Upgrade to Business+ tier = $12.50/user
Year 2 Revenue: 25 × $12.50 × 12 = $3,750

NRR = ($3,750 / $960) × 100% = 391% 🚀
```

### Target Ratios

| Metric | Good | Excellent | Best-in-Class |
|--------|------|-----------|---------------|
| **LTV:CAC** | 3:1 | 5:1 | 10:1 |
| **CAC Payback Period** | 12 months | 6 months | 3 months |
| **Net Revenue Retention (NRR)** | 100% | 110% | 120%+ |
| **Gross Margin** | 70% | 80% | 90% |

## 4. North Star Metric (NSM) Definition & Tracking {#nsm}

### Description

Identifying and relentlessly optimizing the single metric that best captures the core value delivered to customers.

### NSM Selection Framework

#### Characteristics of a Good NSM

1. **Expresses Core Value**: What makes your product indispensable?
2. **Leads Revenue**: Predicts future business success
3. **Actionable**: Teams can influence it directly
4. **Understandable**: Everyone in company grasps it
5. **Measurable**: Can be tracked daily/weekly

#### NSM Examples by Industry

| Company | NSM | Why? |
|---------|-----|------|
| **Spotify** | Total Hours Listened | More listening = more value, higher retention |
| **Airbnb** | Nights Booked | Core transaction, drives revenue |
| **Slack** | Daily Active Teams | Team collaboration is the value |
| **Uber** | Rides Per Week | Frequency = habit formation |
| **WhatsApp** | Messages Sent | Communication is the core value |
| **Netflix** | Content Hours Watched | Engagement = retention |

### Input Metrics (Levers)

Controllable factors that drive your NSM

**Example: Spotify (NSM = Hours Listened)**

```
Input Metrics:
1. New user sign-ups (acquisition)
2. Playlist creation rate (activation)
3. Daily active users (retention)
4. Discovery feature usage (engagement)
5. Offline download rate (habit formation)

If we increase playlist creation by 20%
→ Hours listened increases by 12%
```

### NSM Dashboard Design

**Structure**:

```
┌─────────────────────────────────────────┐
│  North Star Metric: Total Hours Listened │
│        123.5M hours this week (+8%)      │
└─────────────────────────────────────────┘

Input Metrics:
├─ New Sign-Ups: 245K (+12% WoW)
├─ Activation Rate: 68% (vs. 65% target)
├─ Daily Active Users: 12.8M (+5%)
├─ Playlists Created: 1.2M (+15%)
└─ Avg. Session Length: 28 min (-2%)
     ↑ Action Needed: Investigate drop

Cohort Breakdown:
├─ New Users (7-day): 2.1M hours
├─ Active Users (30-day): 95.4M hours
└─ Power Users (90-day): 26M hours
```

### Avoiding Vanity Metrics

**Bad NSM Examples**:

- ❌ Total Sign-Ups (doesn't indicate value delivery)
- ❌ App Downloads (doesn't mean usage)
- ❌ Page Views (can be inflated, not meaningful)

**Good NSM Alternatives**:

- ✅ Weekly Active Users (demonstrates ongoing value)
- ✅ Feature Adoption Rate (shows product stickiness)
- ✅ Customer Health Score (composite of engagement signals)
