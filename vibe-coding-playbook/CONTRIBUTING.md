# Contributing to The Vibe Coding Playbook

We love contributions! Whether you're fixing typos, adding case studies, or expanding sections, all help is welcome.

## How to Contribute

### 1. Fork & Clone

```bash
git clone https://github.com/your-username/vibe-coding-playbook.git
cd vibe-coding-playbook
```

### 2. Create a Branch

```bash
git checkout -b improve/your-topic
```

**Branch naming:**
- `improve/` — Better explanation or structure
- `add/` — New section or guide
- `fix/` — Typo or error
- `case-study/` — New real-world example

### 3. Make Changes

Edit Markdown files directly. Keep it concise and practical.

**Style guide:**
- Use h2 (`##`) for sections, h3 (`###`) for subsections
- Include code examples where helpful
- Add validation checklists at end of sections
- Keep paragraphs short (2–3 sentences)

### 4. Commit & Push

```bash
git add .
git commit -m "improve: clarified token savings section with examples"
git push origin improve/your-topic
```

### 5. Open a Pull Request

Go to GitHub and click "Compare & Pull Request". Describe:
- What you changed
- Why it matters
- Any feedback you'd like

---

## What We're Looking For

✅ **Improvements to existing sections** — Clearer wording, better examples  
✅ **New case studies** — Real projects you've built with Vibe Coding  
✅ **Tool recommendations** — New AI tools, IDEs, or platforms  
✅ **Career path updates** — New roles or resources  
✅ **Bug fixes** — Typos, broken links, outdated info  

❌ **Not looking for:** Promotional content, off-topic additions, unverified claims

---

## Quality Standards

Before submitting, check:

- [ ] Spelling & grammar (use a tool if unsure)
- [ ] Links are correct and current
- [ ] Code examples run without errors
- [ ] Follows style guide above
- [ ] Section includes a validation checklist
