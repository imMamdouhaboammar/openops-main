# Execution Planner Prompt Kit
Pack: `gem-dmm-agent-pack` | Version: 1.0.0 | 2026-02-13

Purpose: Convert a grounded strategy into a phased execution plan with owners, dependencies, and kill/scale rules.

---

## Phasing (default)
### Phase 0 (0–2 weeks): Foundations
Goals:
- tracking + measurement baseline
- creative/landing page minimum set
- experiment backlog ready

Deliverables:
- Measurement spec (events, conversions, dashboards)
- Channel starting budget split (conservative)
- 3–6 experiments ready to run

### Phase 1 (2–6 weeks): Build & Validate
Goals:
- validate channel/offer fit
- identify 1–2 scalable loops
- improve activation and conversion bottlenecks

Deliverables:
- weekly iteration cadence
- paid/SEO/content systems running
- first set of learnings documented

### Phase 2 (6–12 weeks): Scale & Optimize
Goals:
- scale winners
- standardize creative pipeline
- tighten unit economics and attribution

Deliverables:
- budget reallocation rules
- documented playbooks per channel
- risk management and guardrails

---

## Owners and Roles (use placeholders if unknown)
List owners for:
- Paid media
- SEO/content
- Lifecycle/CRM
- CRO
- Analytics/Tracking
- Design/Creative

---

## Dependency Mapping (required)
For each major action include:
- dependency
- blocking vs non-blocking
- mitigation if delayed

---

## Kill / Iterate / Scale Rules (hard requirement)
Define per channel and per experiment:
- Minimum run time and minimum sample size assumptions
- Kill criteria (e.g., CPA > threshold AND no leading indicators improve)
- Iterate criteria (mixed signals)
- Scale criteria (meets target AND guardrails ok)

Guardrails examples:
- unit economics (CAC, payback)
- brand risk (complaints, negative sentiment)
- user experience (bounce rate, conversion drop)
