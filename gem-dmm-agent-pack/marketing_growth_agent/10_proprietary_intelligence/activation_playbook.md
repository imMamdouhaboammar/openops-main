# Classified Intelligence Activation Playbook

**Version**: 1.0.0  
**Last Updated**: January 22, 2026  
**Scope**: Launch Layer 10X+ (Classified) for any new client

---

## Purpose
Provide a step-by-step activation sequence that safely deploys the classified intelligence layer while preserving ethics, privacy, and causal rigor.

---

## Entry Criteria
- Client has defined goals and time horizon
- Access to core data sources or agreed assumptions
- Consent and privacy requirements documented
- Decision owner identified

---

## Required Inputs
- Business model and revenue target
- ICP and audience segments
- Current channel mix and baseline KPIs
- Creative assets inventory
- Proof assets inventory (case studies, data points)
- Constraints (budget, timeline, team)

---

## Phase 0: Intake & Grounding (Day 1-3)

**Actions**
1. Confirm goals, constraints, and risk tolerance
2. Collect baseline KPIs and historical performance
3. Map available data sources and access level

**Outputs**
- Client Context Sheet
- Baseline KPI snapshot
- Data access map

**Gate**
- If baseline data is missing, set conservative assumptions and schedule data QA

---

## Phase 1: Signal Fusion & Trust Audit (Day 3-7)

**Use**: `sovereign_signal_fusion.json`, `trust_signal_ledger.json`, `risk_and_trust_shield.json`

**Actions**
1. Classify signals (first/second/third-party)
2. Assign confidence levels (L1-L4)
3. Audit proof assets for credibility and recency
4. Run privacy and consent checks

**Outputs**
- Signal Fusion Map
- Trust Signal Ledger Scorecard
- Privacy/Compliance Pass

**Gate**
- If confidence < L3 on core KPIs, require causal test plan

---

## Phase 2: Offer Resonance & Archetype Mapping (Day 5-10)

**Use**: `offer_resonance_index.json`, `behavioral_archetype_graph.json`, `message_clarity_entropy.txt`

**Actions**
1. Score offer-market fit using Offer Resonance Index
2. Map primary archetypes and trigger constraints
3. Reduce message entropy to single promise + CTA

**Outputs**
- Offer Resonance Score
- Archetype Map + Trigger Guardrails
- Message Clarity Checklist

**Gate**
- If Offer Resonance < 60, pause scaling and rebuild offer

---

## Phase 3: Creative Genome Seeding (Day 7-14)

**Use**: `creative_genome_engine.json`, `dark_pattern_firewall.json`

**Actions**
1. Encode top-performing creatives into genome schema
2. Generate 6-12 controlled variants (1-2 genes per test)
3. Validate proof/claim compatibility
4. Run dark pattern firewall checks

**Outputs**
- Creative Genome Library
- Variant Test Queue
- Ethical Clearance Log

**Gate**
- No high-risk claims without proof score >= 0.8

---

## Phase 4: Counterfactual & Incrementality Plan (Day 10-18)

**Use**: `counterfactual_growth_simulator.md`, `incrementality_testing_config.json`

**Actions**
1. Build baseline counterfactual model
2. Define incrementality tests (RCT/geo/holdout)
3. Set decision rules and guardrails

**Outputs**
- Counterfactual Baseline
- Incrementality Test Plan
- Decision Rule Sheet

**Gate**
- No budget scaling without causal validation or L3+ confidence

---

## Phase 5: Demand Engineering (Day 12-21)

**Use**: `demand_engineering_system.md`

**Actions**
1. Define category narrative and authority angles
2. Align distribution channels to narrative
3. Connect narrative to measurable funnel metrics

**Outputs**
- Demand Narrative Brief
- Distribution Map
- Funnel Lift Targets

**Gate**
- If narrative does not map to measurable lift, revise messaging

---

## Phase 6: Competitive Wargame & Moat Compounding (Day 14-24)

**Use**: `competitive_wargame_protocol.yml`, `moat_compounding_protocol.txt`

**Actions**
1. Simulate top 3 competitor scenarios
2. Pre-write counter-messaging and budget responses
3. Define moat compounding actions (templates, playbooks)

**Outputs**
- Wargame Response Map
- Counter-Messaging Library
- Moat Compounding Plan

---

## Phase 7: Activation & Guardrailed Scale (Day 18-30)

**Use**: `attention_yield_curve.json`, `risk_and_trust_shield.json`

**Actions**
1. Launch controlled campaigns by channel
2. Apply attention yield curve limits
3. Monitor marginal CAC and fatigue signals
4. Scale only if confidence >= L3 and marginal ROI acceptable

**Outputs**
- Live Campaign Dashboard
- Guardrail Monitoring Log
- Scale/Pause Decisions

---

## Phase 8: Learning Compounding (Ongoing)

**Use**: `compound_learning_orchestrator.yml`

**Actions**
1. Capture learning weekly
2. Update genome, trust ledger, and offer scores monthly
3. Feed improvements into new variants and narratives

**Outputs**
- Learning Digest
- Updated Genome Library
- Budget Optimization Recommendations

---

## Completion Criteria
- Causal test plan executed or in progress
- Creative genome library established
- Offer resonance score >= 60
- Guardrails active and monitored
- Reusable moat assets created
