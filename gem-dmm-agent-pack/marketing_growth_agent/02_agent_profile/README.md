# 02_agent_profile — Super Acting Digital Marketing Expert

**Version:** 2.0.0  
**Date:** 2026-02-13  
**Owner:** OpenOps Studio (Mamdouh Aboammar)

This layer defines the identity and operating contract for the marketing agent in a production-safe way.

## What this folder contains

1. `marketing_expert_profile.json`  
   Canonical machine-readable profile: mission, gates, expertise matrix, evidence model, regional overlays, measurement doctrine, and repo-brain references.

2. `regional_context_egypt_gcc.yaml`  
   Practical operating overlays for Egypt, Saudi, UAE, and GCC-wide campaigns.

3. `communication_contract.yaml`  
   Conversation and delivery contract (max question limits, diagnosis-first behavior, refusal logic, response structure).

4. `profile_operating_manual.md`  
   Human-readable operating guide for operators, prompt engineers, and QA reviewers.

5. `system_prompt_addendum.txt`  
   Drop-in prompt addendum to harden intent-first + touchpoints-driven behavior.

6. `external_grounding_registry_2026.md`  
   External source registry used for grounding (official docs first, GitHub implementation references, Reddit as qualitative-only context).

## Operating intent

- Diagnose first, strategize second.
- Enforce one validated Core Question (CQ).
- Ground important recommendations in independent evidence.
- Adapt strategies for Egypt/GCC market realities without violating privacy or legal boundaries.
