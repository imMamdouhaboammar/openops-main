## 4. Email Sequence Template

**Sequence Name**: [e.g., "Welcome Series"]  
**Type**: [Welcome/Nurture/Re-engagement/Sales]  
**Duration**: [X days]  
**Target Audience**: [Segment]

### Email 1: Welcome
```
Subject Line: [Engaging subject that matches expectation set in landing page]
From Name: [Personal name preferred]
Send Delay: [Immediately or X hours after signup]

Body Structure:
1. Personal greeting and thank you
2. Quick summary of what they're getting
3. Soft CTA: [Link to resource or next step]
4. Signature: [Sender name and info]

Key Elements:
- Mobile optimized (50%+ of opens)
- Plain text feel (not overly designed)
- Link color and CTA button clearly visible
- Unsubscribe link present
```

### Email 2: Establish Value
```
Subject Line: [Benefit-driven subject]
Send Delay: [2 days after Email 1]

Body Structure:
1. Personalized opening (reference their segment/goal)
2. Case study or customer result
3. Specific benefit of your product/service
4. CTA: [Take next step, e.g., "Watch demo"]
5. Alternative CTA: [If not ready, "learn more"]
```

### Email 3: Address Objections
```
Subject Line: [Curiosity or concern address]
Send Delay: [4 days after Email 2]

Body Structure:
1. Common objection/question intro
2. Answer with specific info/data
3. Third-party validation (review, award, testimonial)
4. CTA: [Move to next step]
5. Escape hatch: [Unsubscribe or preference center]
```

### Email 4: Social Proof + Urgency
```
Subject Line: [Social proof or scarcity]
Send Delay: [3 days after Email 3]

Body Structure:
1. Customer success story or stat
2. Specific results achieved
3. Time-limited offer or next cohort info
4. Strong CTA: [Sign up or buy now]
```

### Email 5: Last Chance / Re-engagement
```
Subject Line: [Final call or curiosity angle]
Send Delay: [3 days after Email 4]

Body Structure:
1. Brief recap of benefits
2. Social proof or limited availability
3. Urgency element (deadline, scarcity)
4. Strong CTA: [Last call to action]
5. Alternative for disengaged: [Preference center or soft CTA]
```

## 6. Landing Page Copy Template

### Hero Section
```
Headline: [Primary benefit, benefit-driven or curiosity-based]
- Character count: 40-60 (fits on mobile)
- Keywords: 1-2 target keywords
- Emotional hook: [What feeling does this create?]

Subheadline: [Elaborate on main benefit, add social proof reference if applicable]
- Character count: 60-100
- Expands on headline
- Builds credibility or specificity

CTA Button: [Action-oriented verb + specific outcome]
- Button color: [High contrast with background]
- Button text: "Get [Outcome]" or "Start [Action]" (10-15 characters)
- Location: [Below fold for mobile viewing]
```

### Social Proof Section (3-5 elements)
```
Element 1: [Customer logo, customer name, or specific result]
Element 2: [Stat/metric showing success]
Element 3: [Customer testimonial or award]
Element 4: [Review score or rating]
Element 5: [Certification or partnership]
```

### Problem/Solution Section
```
Problem: [Paint a picture of their current pain]
- 1-2 sentences max
- Specific, not generic
- Emotionally resonant

Solution Intro: "Instead of [old way], you can [new way]"

3 Key Benefits:
1. Benefit #1 (with specific outcome)
2. Benefit #2 (with specific outcome)
3. Benefit #3 (with specific outcome)
```

### Features Section
```
Feature 1: [Name] - [Benefit, not description]
- Icon or image showing feature
- 1-sentence description of what it does
- Result customer achieves

Feature 2: [Name] - [Benefit]
- Icon or image
- 1-sentence description
- Result customer achieves

Feature 3: [Name] - [Benefit]
- Icon or image
- 1-sentence description
- Result customer achieves
```

### Testimonial Section
```
"[Quote highlighting specific result or transformation]"
- Customer Name
- Title / Company
- Photo of customer (if available)
- Specific metric/result mentioned (builds credibility)
```

### FAQ Section (5-7 questions)
```
Q: How long until I see results?
A: [Specific timeframe with example]

Q: What if I'm not sure if it's right for me?
A: [Reassurance statement + trial/guarantee offer]

Q: How much does it cost?
A: [Price or link to pricing page + value statement]

Q: Can I cancel anytime?
A: [Reassurance on commitment / flexibility]
```

### Final CTA Section
```
Headline: [Urgency or benefit reminder - "Last chance" or "Ready to transform?"]
Subheadline: [Final benefit recap or social proof]
Primary CTA Button: [Same as hero CTA or slightly stronger]
Secondary CTA: [Alternative for cold visitors - "See demo" or "Learn more"]
```

## 7. A/B Testing Experiment Template

**Experiment Name**: [Descriptive name]  
**Hypothesis**: [What we're testing and expected outcome]  
**Duration**: [Expected test duration]  
**Minimum Sample Size**: [Calculated based on power analysis]

### Experiment Details
```
Element Testing: [Landing page headline / Email subject / Ad copy / CTA button / etc.]

Control (Version A): [Original version]
Treatment (Version B): [New version being tested]

Expected Impact: [What % improvement are we looking for?]
Confidence Level: [90% or 95% confidence required?]
```

### Hypothesis Statement
```
"If we [change X], then [impact Y] because [reason Z]"

Example:
"If we change the CTA button color from blue to red, then we'll increase conversion rate by 10-15% because red creates more urgency perception."
```

### Sample Size Calculation
```
Baseline conversion rate: [X%]
Minimum detectable effect: [Y%]
Statistical power: [95%]
Significance level: [5%]
Required sample size: [Calculated]
Expected duration: [Days to reach sample size at current traffic]
```

### Metrics to Track
```
Primary Metric: [Conversion rate, CTR, etc.]
Secondary Metrics: [CPC, AOV, LTV, etc.]
Tracking Method: [Google Analytics, platform native, etc.]
```

### Success Criteria
```
Winning version shows: [X% improvement with Y% confidence]
Stopping rule: [If we reach this sample size without significance, we stop]
```

### Results Template (to be filled after test)
```
Winner: [Version A or Version B or Inconclusive]
Improvement: [X% with Y% confidence]
Statistical significance: [Achieved / Not achieved]
Recommendation: [Implement / Run longer / Try new hypothesis]
Next steps: [What to test next based on learnings]
```
