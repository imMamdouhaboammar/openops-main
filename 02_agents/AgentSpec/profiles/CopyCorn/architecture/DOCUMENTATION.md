## 🎯 Executive Overview

### What is CopyCorn v2.0?

CopyCorn v2.0 is a **node-based AI application architecture** that transforms marketing and growth expertise into a composable, scalable system inspired by Google's Opal platform.

Instead of a monolithic AI agent, CopyCorn operates as:

```
20+ Specialized Nodes → Connected Workflows → Modular Outputs → Complete Marketing Systems
```

### Key Innovation

**Traditional Approach**:
```
Mega-Prompt → Black Box Processing → Single Output
```
❌ Hard to debug  
❌ Difficult to customize  
❌ Not reusable  
❌ Limited extensibility

**CopyCorn v2.0 Approach**:
```
Parallel Inputs → Sequential Processing Layers → Composable Outputs → Unified Package
```
✅ Transparent processing  
✅ Highly customizable  
✅ Modular reusability  
✅ Infinite extensibility

## 🏗️ System Architecture

### Conceptual Model

Think of CopyCorn as a **marketing factory**:

```
RAW MATERIALS (Inputs) 
    ↓
PROCESSING LINES (Nodes)
    ↓
FINISHED GOODS (Outputs)
    ↓
PACKAGED PRODUCT (Final System)
```

Each **processing line** (node) has:
- Specific role and expertise
- Defined inputs it requires
- Processing logic (prompt + LLM)
- Structured outputs it produces
- Connections to other nodes

### Data Flow Principles

1. **Parallel Inputs**: All input nodes can be filled simultaneously (no dependencies)

2. **Sequential Layers**: Each processing layer waits for the previous layer to complete

3. **Broadcast Outputs**: One node's output can feed into multiple downstream nodes

4. **Conditional Paths**: Some nodes only execute if specific conditions are met (e.g., Arabic content only if language=AR)

5. **Composable Assembly**: Final output assembles only the requested deliverables

## 🚀 Implementation Guide

### Step 1: Setup (One-Time)

1. **Install Dependencies**:
   ```bash
   # Python environment
   python -m venv venv
   source venv/bin/activate  # or `venv\Scripts\activate` on Windows
   pip install -r requirements.txt
   ```

2. **Configure LLM Access**:
   ```yaml
   # config.yaml
   llm:
     provider: "openai"  # or "anthropic", "google", etc.
     model: "gpt-4-turbo"
     api_key: "your-api-key"
     temperature: 0.7
   ```

3. **Set Up Node Library**:
   ```bash
   # Load all node prompts into vector DB (optional but recommended)
   python scripts/load_prompts.py
   ```

---

### Step 2: Prepare Inputs

1. **Gather Information**:
   - Company/product details
   - Market research
   - Campaign goals
   - Existing assets

2. **Fill Input Forms**:
   - Use templates in `01_input_nodes/`
   - Can be filled in parallel
   - Save as YAML or JSON

3. **Validate Completeness**:
   ```bash
   python scripts/validate_inputs.py --input-dir ./inputs/project_name
   ```

---

### Step 3: Execute Workflow

**Option A: CLI (Full Control)**
```bash
python copycorn.py run \
  --inputs ./inputs/project_name \
  --pattern full \
  --output ./outputs/project_name_2026-01-12
```

**Option B: API (Programmatic)**
```python
from copycorn import CopyCornEngine

engine = CopyCornEngine(config="config.yaml")
result = engine.run(
    inputs_path="./inputs/project_name",
    pattern="full",
    output_path="./outputs/project_name"
)
print(f"Execution ID: {result.execution_id}")
print(f"Status: {result.status}")
print(f"Outputs: {result.output_files}")
```

**Option C: Web UI (User-Friendly)**
```bash
python app.py  # Starts web server on localhost:8080
# Navigate to http://localhost:8080
# Fill forms → Click "Generate" → Download package
```

---

### Step 4: Review Outputs

1. **Check Execution Log**:
   ```bash
   cat outputs/project_name/execution_log.json
   ```

2. **Validate Quality**:
   - Confidence scores per node
   - Completeness metrics
   - Error/warning flags

3. **Browse Deliverables**:
   ```bash
   cd outputs/project_name/CopyCorn_Marketing_System_[Date]
   # Explore folder structure
   ```

---

### Step 5: Iterate & Refine

1. **Feedback Loop**:
   - Mark which outputs need refinement
   - Provide specific feedback
   - Re-run individual nodes if needed

2. **Re-Execute Nodes**:
   ```bash
   python copycorn.py rerun \
     --execution-id abc123 \
     --nodes PR-07,PR-08 \
     --feedback "More contrarian hooks needed"
   ```

3. **Export Final Version**:
   ```bash
   python copycorn.py export \
     --execution-id abc123 \
     --format pdf,docx,notion
   ```

## 🔧 Extension Framework

### Adding New Nodes

CopyCorn is designed for infinite extensibility. Here's how to add new capabilities:

#### Example: Adding "Competitor Analysis" Node

**1. Define Node Specification**

```yaml
# nodes/PR-13_competitor_analysis.yaml

node_id: PR-13
name: "Competitor Analysis"
layer: 2  # Strategy layer
version: 1.0.0

inputs:
  required:
    - IN-02  # Market Context
    - PR-01  # External Research
  optional:
    - IN-04  # Assets (if they have competitor research)

processing:
  type: "llm"
  model: "gpt-4-turbo"
  temperature: 0.7
  max_tokens: 3000
  prompt_path: "prompts/PR-13_competitor_analysis.md"

output_schema:
  type: "json"
  schema_path: "schemas/PR-13_output_schema.json"

connections:
  feeds_into:
    - PR-04  # Direction (inform positioning)
    - PR-06  # Message Architecture (differentiation)

metadata:
  estimated_time: 5  # minutes
  confidence_threshold: 0.7
  tags: ["strategy", "competitive-intelligence"]
```

**2. Write Node Prompt**

```markdown
# prompts/PR-13_competitor_analysis.md

You are the **Competitive Intelligence Specialist** in CopyCorn's squad.

## Inputs
- Market context: {IN-02}
- External research: {PR-01}
- Optional assets: {IN-04}

## Task
Conduct deep competitive analysis:
1. Identify top 3-5 competitors
2. Analyze their positioning & messaging
3. Map their strengths/weaknesses
4. Find gaps to exploit
5. Recommend differentiation strategy

## Output Schema
{schema}
```

**3. Define Output Schema**

```json
// schemas/PR-13_output_schema.json
{
  "competitor_analysis": {
    "competitors": [
      {
        "name": "string",
        "positioning": "string",
        "strengths": ["string"],
        "weaknesses": ["string"],
        "market_share": "string"
      }
    ],
    "gaps_to_exploit": ["string"],
    "differentiation_recommendations": ["string"]
  }
}
```

**4. Register Node**

```python
# copycorn/nodes/registry.py

from copycorn.nodes.base import ProcessingNode

class CompetitorAnalysisNode(ProcessingNode):
    node_id = "PR-13"
    
    def validate_inputs(self, inputs):
        # Custom validation logic
        pass
    
    def process(self, inputs):
        # Load prompt, execute LLM call, validate output
        pass

# Register
NODE_REGISTRY["PR-13"] = CompetitorAnalysisNode
```

**5. Update Workflow**

```yaml
# workflows/full_with_competitor_analysis.yaml

name: "Full Run with Competitor Analysis"
pattern: "extended_full"

layers:
  - layer_1_grounding:
      nodes: [PR-01, PR-02]
  - layer_2_strategy:
      nodes: [PR-03, PR-13, PR-04, PR-05]  # Added PR-13
  # ... rest of workflow
```

**Done!** New node is now part of the system.

## 📚 Best Practices

### For Maximum Quality

1. **Complete Inputs Thoroughly**:
   - Don't skip optional fields if you have the data
   - Provide examples and specifics
   - Include current metrics for context

2. **Review Intermediate Outputs**:
   - Check confidence scores
   - Validate key assumptions
   - Provide feedback if needed

3. **Iterate on Low-Confidence Nodes**:
   - If confidence < 0.7, review and re-run
   - Add more context or constraints
   - Adjust prompts if systematic issues

4. **Combine Automated + Human Expertise**:
   - Use CopyCorn for structure and speed
   - Add your domain expertise on top
   - Refine creative outputs with your taste

---

### For Maximum Efficiency

1. **Use Appropriate Patterns**:
   - Don't run "Full" if you only need creative
   - Match pattern to actual need
   - Skip unnecessary outputs

2. **Batch Similar Projects**:
   - Run multiple clients with same pattern
   - Reuse research across similar industries
   - Create templates for recurring needs

3. **Cache & Reuse**:
   - External research (PR-01) can be cached for 7 days
   - Industry insights apply across clients
   - Save successful experiments as templates

4. **Parallel Execution** (if infrastructure supports):
   - Nodes within same layer can run in parallel
   - Multiple projects can run simultaneously
   - Scale horizontally as needed

---

### For Teams

1. **Standardize Inputs**:
   - Create intake forms for clients
   - Use consistent terminology
   - Build input libraries for common scenarios

2. **Review & Approval Workflows**:
   - Junior team fills inputs
   - CopyCorn generates drafts
   - Senior team reviews and refines
   - Client receives polished deliverables

3. **Knowledge Sharing**:
   - Share successful node outputs
   - Build library of best prompts
   - Document what works per industry

4. **Continuous Improvement**:
   - Track output quality metrics
   - A/B test prompt variations
   - Evolve node prompts based on feedback

## 🔍 Troubleshooting

### Common Issues

#### Issue: Low Confidence Scores

**Symptoms**: Node outputs have confidence < 0.7

**Causes**:
- Insufficient input data
- Ambiguous goals
- Conflicting information
- Limited external research available

**Solutions**:
1. Add more detail to inputs (especially IN-01, IN-03)
2. Provide examples of desired output
3. Run with `--verbose` flag to see research queries
4. Manually supplement with domain knowledge

---

#### Issue: Generic/Irrelevant Outputs

**Symptoms**: Outputs feel cookie-cutter, not specific

**Causes**:
- Inputs too high-level
- Node prompts not contextualized enough
- Missing industry-specific research

**Solutions**:
1. Provide more specific ICP details
2. Include examples of current messaging
3. Add competitor context
4. Customize node prompts for your vertical

---

#### Issue: Processing Takes Too Long

**Symptoms**: Execution > 2 hours

**Causes**:
- LLM API slow/rate-limited
- Complex research queries
- Large output generation

**Solutions**:
1. Use faster model (GPT-3.5 vs GPT-4) for draft
2. Enable caching for research nodes
3. Reduce output quantity (e.g., 25 headlines vs 50)
4. Run in parallel if infrastructure allows

---

#### Issue: Outputs Don't Connect Well

**Symptoms**: Strategy doesn't match tactics, disconnected deliverables

**Causes**:
- Node dependencies not properly configured
- Feedback loop between layers missing

**Solutions**:
1. Check `execution_log.json` for node inputs/outputs
2. Verify all required inputs are present
3. Review node connection mapping
4. Re-run problematic layer with corrected inputs

### Glossary

**Node**: Self-contained processing unit with specific inputs, logic, and outputs

**Layer**: Group of nodes that execute at the same stage of the workflow

**Pattern**: Pre-configured workflow combining specific nodes for a use case

**Grounding**: Research-backed validation before making recommendations

**Growth Loop**: Self-reinforcing system where outputs become new inputs

**Confidence Score**: 0-1 metric indicating node's certainty in its output

**Composable Output**: Modular deliverable that can be used alone or combined
