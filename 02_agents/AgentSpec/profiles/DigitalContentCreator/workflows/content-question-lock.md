# Content Question Lock (CQL) System

## Ensuring Clarity Before Creation

---

## Overview

The Content Question Lock (CQL) is a validation mechanism that ensures we have a clear, measurable objective before creating content. It prevents vague requests and ensures alignment between creator and client.

**Core Principle:** ONE question. ONE clear objective. User-confirmed.

---

## CQL Format

### Standard Template

```
"What [content type] will drive [specific audience] to [specific action] 
on [platform] while maintaining [brand voice]?"
```

### Required Elements

| Element | Description | Examples |
|---------|-------------|----------|
| **Content Type** | Specific format | Caption, post, ad, email, script, article, thread |
| **Specific Audience** | Targetable demographic | "Egyptian women 25-35 interested in skincare" NOT "everyone" |
| **Specific Action** | Measurable behavior | Like, comment, click, buy, sign up, share, download |
| **Platform** | Publishing medium | Instagram, Facebook, LinkedIn, TikTok, Email, Website |
| **Brand Voice** | Tone/personality | Professional, casual, humorous, inspirational, authoritative |

### Optional Elements (Add When Relevant)

| Element | When to Include | Examples |
|---------|-----------------|----------|
| **Key Constraint** | Critical limitation | "...within 150 characters" |
| **Timing** | Time-sensitive | "...for Ramadan campaign" |
| **Product/Service** | Specific offering | "...for our new skincare serum" |
| **Competitive Context** | Positioning | "...differentiating from competitor X" |

---

## Validation Checklist

### ✅ Valid CQL Criteria

- [ ] **Audience is specific enough to target**
  - Can you describe this person in detail?
  - Can you find them on the platform?
  - Do you know their pain points?

- [ ] **Action is measurable**
  - Can you count it?
  - Can you track it?
  - Does the platform support it?

- [ ] **Platform is clearly defined**
  - Specific platform named
  - Format specified (if relevant)
  - Platform capabilities understood

- [ ] **Brand voice is documented**
  - Tone described
  - Examples provided or referenced
  - Consistency with existing content

- [ ] **Constraints are captured**
  - Length limits known
  - Forbidden topics listed
  - Must-include elements noted

- [ ] **Success can be measured**
  - Clear metric defined
  - Baseline known (or acknowledged as unknown)
  - Target set (or range defined)

### ❌ Invalid CQLs (and How to Fix)

**Invalid Example 1: Too Vague**

```
❌ "What should I post?"
```

**Problems:**

- No content type
- No audience
- No action
- No platform
- No voice

**Fixed:**

```
✅ "What Instagram post will drive my target audience to engage?"
```

(Still needs more specificity, but better)

---

**Invalid Example 2: Audience Too Broad**

```
❌ "What post will drive everyone to buy our product?"
```

**Problems:**

- "Everyone" is not targetable
- No platform specified
- No voice defined

**Fixed:**

```
✅ "What Facebook ad will drive Egyptian parents (30-45) with young children 
to purchase our educational toys while maintaining our playful, trustworthy 
brand voice?"
```

---

**Invalid Example 3: Action Not Measurable**

```
❌ "What content will make people like us more?"
```

**Problems:**

- "Like us more" is not measurable
- No content type
- No platform
- No specific audience

**Fixed:**

```
✅ "What Instagram story series will drive our followers to rate us 5 stars 
on Google while maintaining our appreciative, customer-first brand voice?"
```

---

**Invalid Example 4: Missing Critical Context**

```
❌ "Write an Instagram caption"
```

**Problems:**

- No audience
- No action
- No voice
- No objective

**Fixed:**

```
✅ "What Instagram caption will drive fitness enthusiasts (25-40) in Cairo 
to sign up for our free trial class while maintaining our motivational, 
results-focused brand voice?"
```

## CQL Formulation Process

### Step 1: Extract Core Elements

From discovery interview, identify:

- **Who** is the audience?
- **What** do we want them to do?
- **Where** will they see this?
- **How** should we sound?
- **Why** are we creating this?

### Step 2: Draft Initial CQL

Plug elements into template:

```
"What [type] will drive [who] to [what] on [where] while maintaining [how]?"
```

### Step 3: Validate Against Checklist

Run through validation criteria:

- [ ] Audience specific?
- [ ] Action measurable?
- [ ] Platform clear?
- [ ] Voice defined?
- [ ] Constraints captured?
- [ ] Success measurable?

### Step 4: Refine and Confirm

If any criteria fail:

1. Identify missing elements
2. Ask ONLY for missing information (≤5 questions)
3. Update CQL
4. Re-validate

If all criteria pass:

1. Present CQL to user
2. Request confirmation
3. Document in brief
4. Proceed to next stage

## Common CQL Mistakes

### Mistake 1: Multiple Objectives

```
❌ "What post will drive people to like, comment, share, and buy?"
```

**Fix:** Choose ONE primary action

```
✅ "What post will drive people to click the buy link?"
```

### Mistake 2: Unmeasurable Goals

```
❌ "What content will improve brand perception?"
```

**Fix:** Define measurable proxy

```
✅ "What content will drive followers to rate us 5 stars?"
```

### Mistake 3: Platform Mismatch

```
❌ "What LinkedIn post will drive Gen Z to follow us?"
```

**Fix:** Match platform to audience

```
✅ "What TikTok video will drive Gen Z to follow us?"
```

### Mistake 4: Vague Audience

```
❌ "What post will drive people interested in health?"
```

**Fix:** Be specific

```
✅ "What post will drive Egyptian women (30-45) managing diabetes?"
```

## CQL Lock Protocol

### When CQL is Valid

**Agent Response:**

```
✅ Content Question Lock (CQL) Confirmed:

"[CQL statement]"

Success Metric: [metric]
Baseline: [current state or "unknown"]
Target: [goal or range]
Time Horizon: [days/weeks]

This CQL is valid and complete. Proceeding to research and content creation.
```

### When CQL is Invalid

**Agent Response:**

```
⚠️ CQL Incomplete

Current CQL Attempt:
"[partial CQL]"

Missing Elements:
- [element 1]
- [element 2]

To complete the CQL, I need to know:
1. [question]
2. [question]
...

(Maximum 5 questions)
```

### When User Provides Vague Request

**Agent Response:**

```
I want to create the most effective content for you. To do that, I need 
to understand your objective clearly.

Could you help me with:
1. Who is your target audience? (be specific)
2. What action should they take?
3. What platform will this be on?

This will help me formulate a clear Content Question Lock.
```

## Integration with State Machine

### Stage 2: CQL Lock

**Entry Criteria:**
✅ Discovery complete (Stage 1)

**Tasks:**

1. Extract elements from discovery
2. Draft CQL
3. Validate against checklist
4. If invalid → Ask for missing elements (≤5 questions)
5. If valid → Present to user for confirmation
6. Document confirmed CQL

**Exit Criteria:**
✅ CQL formulated  
✅ All required elements present  
✅ User confirmed  
✅ Success metric defined  

**Refusal Rules:**
⛔ Don't proceed without valid CQL  
⛔ Don't create multiple CQLs  
⛔ Don't skip user confirmation
