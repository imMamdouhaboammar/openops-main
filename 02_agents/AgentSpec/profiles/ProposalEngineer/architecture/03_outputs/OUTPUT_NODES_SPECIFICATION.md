# 📤 المخرجات النهائية - OUTPUT NODES

## OUT-01: الملخص التنفيذي (Executive Summary) ⭐

### الغرض
صفحة/صفحتين تحسم كل شيء. 80% من المدراء ما يقرون إلا هذا.

### الهيكل:
```
📌 فتحة قوية (Hook):
"اليوم [الألم برقم] يكلّفك [رقم سنوي] ريال"

📌 الفهم:
"بناءً على فهمنا، التحدي الحقيقي هو [ليس المُعلن]"

📌 الحل:
"لذلك صممنا [3 ركائز أساسية]"

📌 الأثر:
"هذا بيحقق: KPI1 (X > Y خلال Z)، KPI2، KPI3"

📌 ليش نحن:
"بخبرة [سنوات] وسابقة مع جهات مشابهة"

📌 الدعوة:
"نلتزم بـ [التزام واضح] خلال [المدة]"
```

## OUT-02: المقترح الفني الكامل (Technical Proposal) 📄

### المحتويات (13 قسم):

```
1. غلاف جميل بسيط
2. فهرس
3. الملخص التنفيذي (1-2 صفحة)
4. السياق والوضع الراهن (2 صفحة)
5. الأهداف والـ KPIs (1 صفحة)
6. نطاق العمل (1 صفحة)
7. المنهجية (3-4 صفحات)
8. الجدول الزمني (1-2 صفحة)
9. الفريق والحوكمة (1 صفحة)
10. المخرجات (Deliverables) (1-2 صفحة)
11. المخاطر وخطط الطوارئ (2 صفحة)
12. سابقة أعمال (مُعمّاة) (1-2 صفحة)
13. ملاحق (حسب الحاجة)

---
الإجمالي: 18-25 صفحة
```

### معايير الجودة:
- ✅ صفر أخطاء لغوية
- ✅ تنسيق احترافي
- ✅ صور عند الحاجة (لا عشوائية)
- ✅ جداول واضحة
- ✅ لا رابط مكسور

## OUT-03: العرض المالي (Financial Proposal) 💰

### الأقسام:

```
1. ملخص الأسعار
   "الإجمالي: [رقم] ريال"

2. جدول الكميات (BOQ)
   بند 1: إدارة المشروع | [ساعات] | [السعر]
   بند 2: التطوير | [ساعات] | [السعر]
   ... (كل بند واضح)
   
   الإجمالي: ________

3. نسب الدفعات:
   - 20% عند التوقيع
   - 30% بعد الموافقة على الخطة
   - 30% عند الإطلاق
   - 20% عند التسليم

4. الشروط والأحكام:
   - سياسة التسديد
   - سياسة التعديلات
   - سياسة الإلغاء

5. ملاحق:
   - حساب الساعات
   - جدول الخصومات (لو وجد)
   - شروط خاصة
```

### معايير:
- ✅ كل رقم له مبرر
- ✅ الأسعار منطقية مقابل الجهد
- ✅ نسب الدفعات واضحة ومربوطة بالمخرجات

## OUT-04: خطة ضمان الجودة (QA Plan) ✅

### المحتويات:

```
1. معايير الجودة:
   - تقنياً: [المعايير]
   - إدارياً: [المعايير]
   - لغوياً: [المعايير]

2. نقاط الفحص:
   - عند نهاية كل مرحلة
   - عند التسليم النهائي

3. إجراءات التصحيح:
   - خطوات العمل عند اكتشاف مشاكل
   - من يوافق على الحل؟
   - المدة المسموح بها؟

4. سجل المشاكل والحلول:
   جدول: المشكلة | التاريخ | الحل | الحالة
```

## OUT-05: دليل الفريق والحوكمة (Governance Guide) 👥

### المحتويات:

```
1. الهيكل التنظيمي:
   - الرسم البياني
   - الأدوار والمسؤوليات

2. جدول الاجتماعات:
   - يومية/أسبوعية/شهرية
   - المشاركون
   - الأهداف

3. خط التصعيد:
   - المشاكل البسيطة
   - المشاكل المتوسطة
   - المشاكل الحرجة

4. نقاط الاتصال:
   - مدير المشروع الرئيسي
   - مدير فريق التطوير
   - ضابط الجودة
   - ضابط المخاطر

5. معايير الاتصال:
   - ساعات الاستجابة
   - قنوات الاتصال (بريد/هاتف/رسائل)
```


## OUT-07: ملخص تنفيذي (2 دقيقة—قراءة) 📝

### الصيغة (8 نقاط):

```
1. الألم/الفرصة برقم واحد يوجّع
   "زمن الموافقات 72 ساعة"

2. الحل بجملتين
   "[الحل A] و [الحل B]"

3. ثلاث KPIs بزمن
   "من X > إلى Y خلال Z"

4. نطاق داخل/خارج
   "تشمل: [...], لا تشمل: [...]"

5. ثلاث مراحل رئيسية
   "مرحلة 1: [...], مرحلة 2: [...], مرحلة 3: [...]"

6. فريق وحوكمة
   "مدير: [الاسم], موعد الاجتماع: [...]"

7. ثلاث مخاطر وخطة B
   "الخطر 1: [...], الحل: [...]"

8. إنجاز مشابه مُعمّى
   "جهة حكومية: حققنا خفض 57% خلال 4 أيام"
```

## معايير الإخراج الموحدة

### 🎨 التصميم والتنسيق:
- لون أساسي + لون ثانوي
- خط عنوان واضح
- هوامش متساوية (2.5 سم)
- رقم الصفحات أسفل اليمين

### 📝 اللغة:
- عربية سعودية احترافية
- مصطلحات موحدة
- لا نسخ من عروض سابقة
- تدقيق 100%

### 🔢 الأرقام:
- كل رقم له مصدر واضح
- أرقام واقعية (ليست متفائلة جداً)
- أسعار شفافة

### 📋 الهيكل:
- فهرس بسيط
- عناوين واضحة
- قوائم بدل فقرات طويلة
- جداول عند الحاجة

### ✅ نقاط التحقق:
- عرض الملخص لمدير المشروع قبل الإرسال
- مراجعة قانونية إن لزم الأمر
- تصحيح إملائي نهائي
- طباعة تجريبية للتأكد من الجودة
