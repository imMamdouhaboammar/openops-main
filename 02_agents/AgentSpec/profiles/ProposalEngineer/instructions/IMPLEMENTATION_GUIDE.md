# 📚 دليل التنفيذ والإرشادات

## 🎯 ابدأ من هنا

اختر النمط المناسب لحالتك:

### 🚀 Mode 1: العرض الكامل (FULL MODE)
**المدة**: 72 ساعة
**الاستخدام**: عرض شامل محترف

```
ساعة 0-24:
- جمع البيانات الكاملة
- تحليل المشكلة
- تحديد KPIs

ساعة 24-48:
- كتابة 13 قسم
- تصميم الجداول
- تنسيق أولي

ساعة 48-72:
- تدقيق لغوي شامل
- مراجعة جودة
- إرسال نهائي
```

### ⚡ Mode 2: العرض السريع (EXPRESS MODE)
**المدة**: 24-30 ساعة
**الاستخدام**: مهلة ضيقة لا تسمح بالتفاصيل

```
ساعة 0-8:
- جمع البيانات الضرورية فقط
- فهم سريع للمشكلة

ساعة 8-20:
- كتابة 10 أقسام (حذف التفاصيل الزائدة)
- تنسيق أساسي

ساعة 20-30:
- تدقيق سريع
- إرسال
```

### 📄 Mode 3: ملخص الإدارة (EXECUTIVE BRIEF MODE)
**المدة**: 4-6 ساعات
**الاستخدام**: فقط 2-3 صفحات للمدراء

**المحتوى**:
- صفحة واحدة ملخص تنفيذي
- صفحة واحدة السياق والأهداف
- صفحة واحدة النطاق والفريق

### 💰 Mode 4: الأسعار فقط (FINANCIAL ONLY MODE)
**المدة**: 8-12 ساعة
**الاستخدام**: تقديم أسعار على عرض فني معروف

**المحتوى**:
- ملخص الأسعار
- BOQ كامل
- نسب الدفعات
- الشروط والأحكام

### 🔄 Mode 5: تحديث/تعديل (AMEND MODE)
**المدة**: 4-8 ساعات
**الاستخدام**: تعديل عرض قائم بناءً على تعليقات

**الخطوات**:
- فهم التعديلات المطلوبة
- تطبيق التعديلات
- تدقيق الأقسام المتأثرة فقط
- إرسال النسخة الجديدة

## 🔧 نصائح عملية

### ✅ للملخص التنفيذي:
- ركّز على رقم واحد يوجّع الآن
- اكتب بأسلوب قصير وحاسم
- لا تكرّر ما في الأقسام الأخرى

### ✅ للأسعار:
- لا تخفِ الحسابات
- كل رقم يجب يكون منطقي ومبرر
- أضف 10% هامش مخاطرة على الأقل

### ✅ للمخاطر:
- تعترف بها = ثقة وليس ضعف
- لكل مخطرة = خطة بديلة واضحة
- لا تتجاهل أي مخطرة

### ✅ للتدقيق اللغوي:
- اقرأ النص بصوت عالي (تسمع الأخطاء)
- اطلب شخص آخر يراجع
- استخدم أدوات: Grammarly / Diacritic
- الأخطاء ممنوعة 100%

| ❌ الخطأ | ✅ الحل |
|---------|---------|
| كلام عام بدون أرقام | "من 72 ساعة > إلى 24 ساعة خلال 30 يوم" |
| تجاهل النطاق | "داخل: A,B,C • خارج: X,Y,Z" |
| أسعار بلا تفصيل | BOQ كامل لكل بند |
| لا مخاطر محددة | 3 مخاطر + خطة بديلة |
| تدقيق ناقص | 100% خالي من الأخطاء |
| نسخ من عروض قديمة | أعد الصياغة تماماً |

## 📞 نقاط الدعم

### عند الارتباك:
1. ارجع إلى [MAIN_SYSTEM_PROMPT.md](MAIN_SYSTEM_PROMPT.md)
2. اقرأ المبدأ ذو الصلة
3. انظر لـ TEMPLATES_AND_CHECKLISTS.md مثال

### عند النقد أو التعديلات:
1. اسأل: "هذا التعديل في النطاق الأصلي أم إضافة جديدة؟"
2. إن كانت إضافة = تعديل رسمي + سعر إضافي
3. إن كانت تصحيح = احسبها جزء من الجودة

### عند تأخر العميل:
1. ذكّر بالتواريخ المتفق عليها
2. إذا تجاوز بـ 5 أيام = أضف تأخير في الجدول
3. وثّق كل شيء
