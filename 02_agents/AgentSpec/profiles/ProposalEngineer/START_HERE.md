# 🚀 ابدأ هنا - Proposal Engineer

## وش هذا النظام؟

هذا نظام متخصص في **بناء عروض فنية ومالية احترافية** تحسم القرار وتفوز بالعقود.

### الفرق؟
- ❌ عروض عادية = كتابة جميلة بدون أرقام
- ✅ عروضنا = **هندسة قرار** مع أرقام واضحة

---

## 5 أنماط عمل سريعة

| Mode | المدة | الاستخدام |
|------|-------|-----------|
| **Full** | 72 ساعة | عرض شامل 20-30 صفحة |
| **Express** | 24-30 ساعة | عرض مركّز 10-12 صفحة |
| **Executive Brief** | 4-6 ساعات | ملخص إدارة فقط 2-3 صفحات |
| **Financial Only** | 8-12 ساعة | العرض المالي فقط |
| **Amend** | 4-8 ساعات | تعديل عرض قائم |

---

## 15 خطوة—من الفكرة للتسليم

### المرحلة الأولى: التقييم ⏱️ (10 دقائق)
**1️⃣ قرار Bid/No-Bid**
- هل عندنا وقت وناس؟ وقت منطقي؟ عندنا الخبرة؟
- **النتيجة**: ندخل أو نسحب

---

### المرحلة الثانية: الفهم 🔍 (72 ساعة)

**2️⃣ INPUT_01**: السياق التجاري
- الهدف الحقيقي؟ الميزانية؟ الجهات؟ المهلة؟

**3️⃣ INPUT_02**: فهم الألم
- وش التحدي الحقيقي؟ وش الأرقام اللي توجّع؟

**4️⃣ INPUT_03**: المتطلبات
- كراسة الشروط (RFP)؟ المتطلبات الفنية؟ معايير القبول؟

**5️⃣ INPUT_04**: صاحب القرار
- من يوقّع؟ من يراجع؟ وش أولوياته؟

---

### المرحلة الثالثة: البناء 🏗️ (24-48 ساعة)

**6️⃣ الملخص التنفيذي** (صفحة/صفحتين)
- الألم برقم > الحل بجملتين > 3-5 KPIs بزمن

**7️⃣ السياق والوضع الراهن**
- الأرقام اللي توجّع

**8️⃣ الأهداف والـ KPIs**
- من X > إلى Y خلال Z (مصدر القياس)

**9️⃣ النطاق (Scope)**
- ماتريكس: داخل/خارج/مشروط

**🔟 المنهجية**
- 4 مراحل: اكتشاف > تصميم > تنفيذ > تسليم

**1️⃣1️⃣ الجدول الزمني**
- تواريخ فعلية + نسب جهد

**1️⃣2️⃣ الفريق والحوكمة**
- نقطة اتصال + خط تصعيد

**1️⃣3️⃣ المخرجات (Deliverables)**
- قائمة محددة + معايير قبول

**1️⃣4️⃣ المخاطر وخطة B**
- 3 مخاطر على الأقل + تخفيف واضح

**1️⃣5️⃣ التبعيات والافتراضات**
- وش نحتاج من العميل؟

---

### المرحلة الرابعة: الأسعار ✅ (24 ساعة)

**1️⃣6️⃣ العرض المالي**
- BOQ (جدول الكميات) تفصيل كامل
- نسب الدفعات مربوطة بالمخرجات
- هامش مخاطرة (10-15%)

**1️⃣7️⃣ بار الجودة**
- شيّك قائمة التحقق 10 نقاط
- إرسال نهائي

---

## 10 مبادئ ذهبية

| # | المبدأ | الملاحظة |
|---|--------|----------|
| 1️⃣ | افهم المرض أولاً | ليش الجهة تبغى هذا الآن؟ وش يخوفهم؟ |
| 2️⃣ | افصل: تفكير + تنفيذ + فلوس | 3 طبقات مستقلة، لكن متوازنة |
| 3️⃣ | كل بند = مخرج واضح + توقيت | ممنوع كلام عام |
| 4️⃣ | الملخص يحسم كل شيء | 80% من المدراء ما يقرون إلا الصفحة الأولى |
| 5️⃣ | المخاطر = ثقة | الاعتراف بها يبان أنك شريك نجاح |
| 6️⃣ | الأسعار انعكاس للفني | من الوقت > الجهد > الموارد > المخاطر |
| 7️⃣ | الوضوح = الفوز | لغة هادئة واثقة بدون استعراض |
| 8️⃣ | السرية أولاً (NDA) | لا أسماء عملاء + أرقام مجمّعة |
| 9️⃣ | النطاق الواضح = حماية | "ما يشمل: [قائمة محددة]" |
| 🔟 | لا أخطاء لغوية | الأخطاء مرفوضة 100% |

## أخطاء شائعة وتصحيحاتها

### ❌ كلام عام
```
"سنحسن الكفاءة والإنتاجية"
```
✅ **البديل الصحيح**:
```
"سننقل زمن الموافقات من 72 ساعة إلى 24 ساعة خلال 30 يوم"
```

### ❌ KPI بدون مدة
```
"سنرفع الرضا"
```
✅ **البديل الصحيح**:
```
"من 65% > إلى 90% خلال 4 أسابيع (أداة المراجعة)"
```

### ❌ خلط الأسعار في الفني
```
"تطوير المنصة: 500,000 ريال"
```
✅ **البديل الصحيح**:
```
الفني: "غير الأسعار"
المالي: "بند A: 500,000 ريال | تفاصيل: [...]"
```
